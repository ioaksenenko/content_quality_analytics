3.3.3 Подчинение

Пусть P ,   Q  – процессы, такие, что α P ⊆ α Q . В комбинации ( P   | |   Q )  каждое действие P  может произойти, только если это позволяет Q , тогда как Q  может независимо осуществлять действия из ( α P   –   α Q )  без разрешения партнера P . Таким образом, P  служит по отношению к Q  подчиненным процессом, тогда как Q  выступает как главный процесс. Это записывается так: P   //   Q . Отсюда: α   ( P   //   Q ) = ( α P   –   α Q ) .

Обычно подчиненному процессу дается имя (например, m  ), которое используется главным процессом для всех взаимодействий с подчиненным. Каждое взаимодействие по каналу c  обозначается тройкой m . c . v , где α m . c   ( m : P )   =   α c   ( P ) ,   а   v ∈ α c   ( P ) .

В конструкции ( m : P   //   Q )  процесс Q  взаимодействует с  P по каналам с составными именами вида m . c , тогда как P  для этих же взаимодействий использует соответствующий простой канал c . Так как все эти взаимодействия скрыты от обстановки, имя m  недоступно снаружи; следовательно, оно служит локальным именем подчиненного процесса.

Подчинение может быть вложенным.

Пример 3.25

( n : ( m : P   //   Q )   //   R ) .

Процесс R  не имеет возможности ни непосредственно взаимодействовать с  P , ни знать о существовании P  и его имени m .

Пример 3.26

у д в : У Д В   //   Q .

Подчиненный процесс ведет себя как обыкновенная подпрограмма, вызываемая внутри главного процесса Q . Внутри Q  значение 2   ×   e  может быть получено поочередным выводом аргумента e  по левому каналу процесса удв и вводом результата по правому каналу:

у д в . л е в ! e → ( у д в . п р а в ? x →   … ) .

Пример 3.27

с т : С Т Е К   //   Q .

Внутри главного процесса Q . с т . л е в ! v  используется для проталкивания в верхушку стека, а с т . п р а в ? v  выталкивает верхнее значение. Использование конструкции выбора позволяет рассматривать ситуацию, когда стек пуст:

( с т . п р а в ? x → Q 1 ( x )   | с т . п у с т → Q 2 ) .

Если стек не пуст, то выбирается первый путь; если пуст, то выбор второго пути позволяет избежать тупика.

Оператор подчинения может использоваться для рекурсивного определения подпрограмм. Каждый уровень рекурсии (кроме последнего) задает новую локальную подпрограмму, работающую с рекурсивным вызовом.

Ф А К Т = μ X . л е в ? n   → ( i f   n = 0   t h e n ( п р а в ! → x ) ) e l s e   ( f : X   //   ( f . л е в ( n   –   1 ) → f . п р а в ? y → п р а в ! ( n   ×   y ) → X ) ) .

Программа Ф А К Т использует каналы л е в и п р а в для обмена результатами и параметрами с вызывающим ее процессом, а каналы f . л е в  и f . п р а в  – для взаимодействия со своим подчиненным процессом f .
