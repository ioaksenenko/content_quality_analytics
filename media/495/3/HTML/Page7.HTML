3.1.5 Операции над протоколами

Протоколам принадлежит основная роль в фиксировании, описании и понимании поведения процессов. Введем следующие обозначения:

  • s ,   t ,   u  – протоколы;
  • S ,   T ,   U  – множества протоколов;
  • f ,   g ,   h  – функции.

Конкатенация. Наиболее важной операцией над протоколами является конкатенация s ∧ t , которая строит новый протокол из пары протоколов s  и  t , просто соединяя их в указанном порядке.

Пример 3.12

Пример конкатенации:

〈 м о н ,   ш о к 〉 ∧ 〈 м о н 〉   =   〈 м о н ,   ш о к ,   м о н 〉 .

Самые важные свойства конкатенации – это ее ассоциативность и то, что пустой протокол 〈 〉  служит для нее единицей:

s ∧ ( t ∧ u ) = ( s ∧ t ) ∧ u ;

s ∧ 〈 〉 = 〈 〉 ∧ s = s .  

Пусть f  – функция, отображающая протоколы в протоколы. Будем говорить, что функция строгая, если она отображает пустой протокол в пустой протокол: f ( 〈 〉 ) = 〈 〉 .

Будем говорить, что функция f  дистрибутивна, если f ( s ∧ t )   = f ( s ) ∧ f ( t ) . Все дистрибутивные функции являются строгими.

Если n  – натуральное число, то t n  будет обозначать конкатенацию n  копий протокола t . Отсюда следует:

t n   +   1   =   t ∧ t n ;

( s ∧ t ) n   +   1   =   s ∧ ( s ∧ t ) n   ∧ t .

Сужение. Выражение ( t   ↑ A )  обозначает протокол t , суженный на множество символов A ; он строится из t  отбрасыванием всех символов, не принадлежащих A .

Сужение дистрибутивно и поэтому строго:

〈 〉 ↑ A = 〈 〉 ;

( s ∧ t ) ↑ A = ( s ↑ A ) ∧ ( t ↑ A ) .

Эффект сужения на одноэлементных последовательностях очевиден:

〈 x 〉 ↑ A = 〈 x 〉 ,   если   x ∉ A ;

〈 y 〉 ↑ A = 〈 〉 ,   если   y ∉ A .

Приведенные ниже законы раскрывают взаимосвязь сужения « ↑ »  и операций над множествами:

s ↑ ∅ = 〈 〉 ;

( s ↑ A ) ↑ B = s ↑ ( A ∪ B ) .

Голова и хвост. Если s  – непустая последовательность, обозначим ее первый элемент s 0 , а результат, полученный после его удаления, – s ′ . Например, 〈 x ,   y ,   x 〉 0 = x ,   〈 x ,   y ,   x 〉 ′ = y . Обе эти операции не определены для пустой последовательности:

( 〈 x 〉 ∧ s ) 0 = x ;

( 〈 x 〉 ∧ s ) ′ = s ;

s = ( 〈 s 0 〉 ∧ s ′ ) ,   если   s ≠ 〈 〉 .

Закон равенства или неравенства двух протоколов:

s = t ≡ ( s = t = 〈 〉   O R   ( s 0 =   t 0   A N D   s ′ = t ′ ) ) .

Звездочка. Множество A ∗  – это набор всех конечных протоколов (включая 〈 〉 ), составленных из элементов множества A . После сужения на A  такие протоколы остаются неизменными, Отсюда следует простое определение:

A ∗ =   { s | ( s ↑ A )   = s } .

Следствиями этого определения являются законы:

〈 〉 ∈ A ∗ ;

〈 x 〉 ∈ A ∗ ≡ x ∈ A ;

( s ∧ t ) ∈ A ∗ ≡ s ∈ A ∗   A N D   t ∈ A ∗ .

Они обладают достаточной мощностью, чтобы определить, принадлежит ли протокол множеству A ∗ .

Пример 3.13

Если x ∈ A , а y ∉ A , то

〈 x ,   y 〉 ∈ A ∗ ≡ ( 〈 x 〉 ∧ 〈 y 〉 ) ∈ A ∗ ≡ ( 〈 x 〉 ∈ A ∗ )   A N D   ( 〈 y 〉 ∈ A ∗ ) = T   A N D   F = F .

Порядок. Если s  – копия некоторого начального отрезка t , то можно найти такое продолжение u  последовательности s , что s ∧ u   =   t . Определим отношение порядка:

s ≤ t = ( ∃ u . s ∧ u = t )

и будем говорить, что s является префиксом  t . Например: 〈 x ,   y 〉 ≤   〈 x ,   y ,   z 〉 . Отношение « ≤ »  является частичным упорядочением и имеет своим наименьшим элементом 〈 〉 . Об этом говорят законы:

  • 〈 〉 ≤ s  – наименьший элемент;
  • s ≤ s  – рефлексивность;
  • s ≤ t   A N D   t ≤ s ⇒ t = s  – антисимметричность;
  • s ≤ t   A N D   t ≤ u ⇒ s ≤ u  – транзитивность.

Следующий закон позволяет определить, является ли справедливым отношение s   ≤ t :

( 〈 x 〉 ∧ s ) ≤ t ≡ t ≠   〈 〉   A N D   x = t 0   A N D   s ≤ t ′ .

Будем говорить, что функция f  из множества протоколов во множество протоколов монотонна, если она сохраняет отношение порядка « ≤ » , т. е. f ( s ) ≤ f ( t )  всякий раз, когда s   ≤ t .

Длина. Длину протокола t  будем обозначать # t . Например, #   〈 x ,   y ,   z 〉 = 3 . Следующие законы определяют операцию « # » :

#   〈 〉 = 0 ;

#   〈 x 〉 = 1 ;

#   〈 s ∧ t 〉 = # s   +   # t .

Число вхождений символа x  в протокол s  определяется как s ↓ x =   #   ( s ↑ { x } ) .
